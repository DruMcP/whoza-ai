# Rex Technical Implementation Guide

## System Architecture

### Core Components

1. **Database Layer** (`supabase/migrations/create_rex_decision_engine_v2.sql`)
   - 5 main tables with RLS policies
   - Helper functions for common operations
   - Time-series tracking of confidence scores

2. **Decision Engine** (`src/services/rexDecisionEngine.js`)
   - Business evaluation logic
   - Recommendation generation
   - Pillar scoring algorithms
   - Action completion tracking

3. **UI Components** (`src/components/Rex*.jsx`)
   - `RexDashboard` - Main interface
   - `RexEvaluationDisplay` - Shows 5 pillar scores
   - `RexRecommendationCard` - Active recommendation display
   - `RexActionHistory` - Completed actions log
   - `RexConfidenceTrend` - Score visualization

4. **Integration** (`src/pages/Portal.jsx`)
   - Rex tab in portal navigation
   - Seamless access to Rex features

## Database Schema

### rex_evaluation_criteria
Stores configurable criteria for ECE assessment.

**Key Fields:**
- `pillar` - Which ECE pillar (entity_clarity, consensus_shaping, etc.)
- `criterion_key` - Unique identifier
- `criterion_name` - Human-readable name
- `description` - What it evaluates
- `weight` - Importance factor (1-5)

**Seeded Data:** 20 default criteria across all 5 pillars

### rex_ece_evaluations
Stores full business evaluations with pillar scores.

**Key Fields:**
- `business_id` - Link to business profile
- `entity_clarity_score` - 0-100
- `consensus_score` - 0-100
- `answer_readiness_score` - 0-100
- `risk_reduction_score` - 0-100
- `confidence_score` - 0-100
- `overall_score` - Computed average (generated column)
- `weakest_pillar` - Target for next recommendation

**Notes:**
- Each score has associated notes and arrays of issues
- `evaluation_data` stores full business snapshot
- Historical record of all evaluations

### rex_recommendations
Stores recommendation lifecycle (ONE active per business).

**Key Fields:**
- `business_id` - Link to business
- `target_pillar` - Which pillar this addresses
- `action_type` - Internal classification
- `title` - User-facing recommendation title
- `explanation` - Why this matters
- `exact_copy` - Copy-paste text (nullable)
- `application_location` - Where to apply
- `expected_impact` - What to expect
- `status` - Current state (draft → completed)

**Status Flow:**
1. `draft` - Generated by Rex
2. `sent_to_admin` - Internal review
3. `sent_to_customer` - Awaiting user approval
4. `approved` - Ready to implement
5. `in_progress` - User working on it
6. `completed` - Done
7. `declined` - User rejected

**Critical Constraint:**
Partial unique index ensures only ONE active recommendation per business:
```sql
CREATE UNIQUE INDEX idx_rex_recommendations_one_active_per_business
ON rex_recommendations(business_id)
WHERE status IN ('draft', 'sent_to_admin', 'sent_to_customer', 'approved', 'in_progress');
```

### rex_action_history
Permanent record of completed actions and their impact.

**Key Fields:**
- `recommendation_id` - Link to original recommendation
- `target_pillar` - Which pillar was addressed
- `what_changed` - User description of change
- `where_applied` - Where change was made
- `confidence_before` / `confidence_after` - Score delta
- `pillar_score_before` / `pillar_score_after` - Pillar-specific delta

**Use Cases:**
- Impact tracking
- Historical analysis
- User activity log
- Confidence trend calculation

### rex_confidence_scores
Time-series snapshots of confidence scores.

**Key Fields:**
- `entity_clarity_score` through `confidence_score` - All 5 pillars
- `overall_confidence` - Overall score
- `measurement_type` - Why this was recorded (initial, post_action, monthly_review, manual)
- `triggered_by_action_id` - Link to action that caused measurement
- `measured_at` - Timestamp

**Measurement Types:**
- `initial` - First evaluation
- `post_action` - After completing an action
- `monthly_review` - Scheduled re-evaluation
- `manual` - Admin-triggered

## Rex Decision Engine API

### Core Methods

#### `evaluateBusiness(businessId, userId)`
Performs full ECE evaluation across all 5 pillars.

**Returns:** Evaluation object with all scores

**Side Effects:**
- Inserts record into `rex_ece_evaluations`
- Records initial confidence score
- Stores business data snapshot

**Algorithm:**
1. Gather business data (profile, tasks, scores)
2. Evaluate each pillar independently
3. Identify weakest pillar
4. Calculate overall score
5. Store results

#### `generateRecommendation(businessId, userId, evaluationId)`
Generates ONE recommendation for weakest pillar.

**Returns:** Recommendation object

**Validation:**
- Throws error if active recommendation already exists
- Ensures only ONE recommendation per business

**Algorithm:**
1. Get latest evaluation (or run new one)
2. Check for existing active recommendations
3. Identify weakest pillar
4. Generate pillar-specific action
5. Insert recommendation with status 'draft'

#### `completeAction(recommendationId, completionNotes, actualImpact)`
Records action completion and triggers re-evaluation.

**Returns:**
- Action history record
- New evaluation
- Score increase

**Process:**
1. Fetch original recommendation and evaluation
2. Update recommendation status to 'completed'
3. Re-evaluate business
4. Calculate score delta
5. Insert action history record
6. Record new confidence score

#### `updateRecommendationStatus(recommendationId, status, additionalData)`
Updates recommendation workflow state.

**Statuses:**
- `sent_to_admin` - Sets `sent_to_admin_at`
- `sent_to_customer` - Sets `sent_to_customer_at`
- `approved` - Sets `approved_at`
- `completed` - Sets `completed_at`
- `declined` - Sets `declined_at`

### Evaluation Algorithms

#### Entity Clarity Scoring
**Base Score:** 50
**Deductions:**
- Missing business name: -20
- Missing industry: -15
- Missing location: -15
- Generic terms in name: -10
**Bonuses:**
- All core fields complete: +20

#### Consensus Shaping Scoring
**Base Score:** 60
**Deductions:**
- No website: -20
- Description < 50 chars: -15
**Bonuses:**
- Description >= 100 chars: +15

#### Answer Readiness Scoring
**Base Score:** 40
**Deductions:**
- Description < 100 chars: -15
- No conversational language: -10
- No services listed: -15
**Bonuses:**
- Conversational format: +20
- Services listed: +30

#### Risk Reduction Scoring
**Base Score:** 50
**Deductions:**
- No phone: -15
- No email: -10
- No website: -20
- No credentials: -10
**Bonuses:**
- All contact info present: +35

#### Confidence Accretion Scoring
**Base Score:** 45
**Deductions:**
- No recent tasks: -15
- No score history: -10
- Negative trend: -5
- Profile < 70% complete: -10
**Bonuses:**
- Has recent tasks: +20
- Positive trend: +25

### Recommendation Generation Logic

Each pillar has a dedicated generator function:

```javascript
generateEntityClarityAction(profile, evaluation)
generateConsensusShapingAction(profile, evaluation)
generateAnswerReadinessAction(profile, evaluation)
generateRiskReductionAction(profile, evaluation)
generateConfidenceAccretionAction(profile, evaluation)
```

**Generator Priority:**
1. Check evaluation weaknesses/conflicts/gaps arrays
2. Select most critical issue
3. Generate specific, actionable recommendation
4. Include exact copy when possible
5. Estimate time (typically 2-5 minutes)

**Recommendation Templates:**
Each generator has 3-4 recommendation templates for common issues. Selection is based on weakness priority.

## Development Workflow

### Adding a New Evaluation Criterion

1. Insert into `rex_evaluation_criteria`:
```sql
INSERT INTO rex_evaluation_criteria (pillar, criterion_key, criterion_name, description, weight)
VALUES ('entity_clarity', 'new_criterion', 'New Check', 'What this evaluates', 3);
```

2. Update relevant evaluation function in `rexDecisionEngine.js`:
```javascript
evaluateEntityClarity(profile) {
  // Add logic for new criterion
  if (!profile.newField) {
    weaknesses.push('New criterion not met');
    score -= 10;
  }
}
```

### Adding a New Recommendation Type

1. Add to appropriate generator function:
```javascript
generateEntityClarityAction(profile, evaluation) {
  const weaknesses = evaluation.entity_clarity_weaknesses || [];

  // New recommendation type
  if (weaknesses.some(w => w.includes('new_issue'))) {
    return {
      action_type: 'new_action_type',
      title: 'Action Title',
      explanation: 'Why this matters',
      exact_copy: 'Copy-paste text or null',
      application_location: 'Where to apply',
      expected_impact: 'Expected result',
      estimated_minutes: 5
    };
  }

  // Existing recommendations...
}
```

5. Generate recommendation

## Security Considerations

### Row Level Security
All Rex tables enforce RLS:
- Users can only see their own data
- Admins can view all data
- No public access

### Data Privacy
- Business data snapshots stored in `evaluation_data` (JSONB)
- Consider GDPR implications for EU customers
- Recommendation text may contain business-specific info

### Input Validation
- All user inputs sanitized by Supabase
- Status transitions validated server-side
- ONE active recommendation constraint enforced at DB level

## Performance Optimization

### Database Indexes
Key indexes created for common queries:
- `idx_rex_evaluations_business` - Fast lookup by business
- `idx_rex_recommendations_status` - Filter by status
- `idx_rex_history_completed` - Sort by completion date
- `idx_rex_confidence_measured` - Time-series queries

### Query Optimization
- Use `maybeSingle()` for zero-or-one results
- Parallel data loading in `loadRexData()`
- Computed columns for overall scores (no runtime calculation)

### Caching Strategy (Future)

## Troubleshooting

### "Active recommendation already exists"
**Cause:** Attempting to generate new recommendation when one is pending

**Solution:** Complete or decline existing recommendation first

**Query to check:**
```sql
SELECT * FROM rex_recommendations
WHERE business_id = 'xxx'
AND status IN ('draft', 'sent_to_admin', 'sent_to_customer', 'approved', 'in_progress');
```

### Score not increasing after action
**Cause:** Change may not address evaluation criteria

**Solution:**
1. Check completion notes
2. Re-run evaluation manually
3. Verify change was actually made

### Missing business profile
**Cause:** Rex requires a business profile to evaluate

**Solution:** Ensure user has completed business profile setup first
