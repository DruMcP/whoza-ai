# 5R-AI Brand Style Guide

## Brand Overview

5R-AI is a premium AI-powered service designed for UK tradespeople to improve their visibility in AI search results. The brand identity balances professionalism and approachability, reflecting both cutting-edge technology and practical utility for working professionals.

## Typography

### Font Families

**Primary Font: Inter**
- Used for: Body text, UI elements, forms
- Weights: 400 (regular), 500 (medium), 600 (semibold), 700 (bold)
- Features: Clean, highly legible, optimized for screens
- Fallback: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', sans-serif

**Display Font: Lexend**
- Used for: Headings, numbers, emphasized text
- Weights: 400 (regular), 500 (medium), 600 (semibold), 700 (bold)
- Features: Modern, geometric, excellent readability
- Purpose: Creates visual hierarchy and premium feel

### Typography Scale

#### Headings

**H1 - Hero Headlines**
- Font: Lexend
- Size: 56px (desktop), 36px (mobile)
- Weight: 800
- Line Height: 1.1
- Letter Spacing: -0.03em
- Color: Neutral 900
- Use: Landing page heroes, major section titles

**H2 - Section Headlines**
- Font: Lexend
- Size: 40px (desktop), 28px (mobile)
- Weight: 700
- Line Height: 1.2
- Letter Spacing: -0.02em
- Color: Neutral 900
- Use: Page sections, major content divisions

**H3 - Subsection Titles**
- Font: Lexend
- Size: 28px (desktop), 22px (mobile)
- Weight: 600
- Line Height: 1.3
- Letter Spacing: -0.02em
- Color: Primary 800 or Neutral 900
- Use: Cards, subsections, feature titles

**H4 - Component Titles**
- Font: Lexend
- Size: 20px
- Weight: 600
- Line Height: 1.4
- Letter Spacing: -0.02em
- Use: Small component headings, labels

#### Body Text

**Body Large**
- Font: Inter
- Size: 21px
- Line Height: 1.6
- Color: Neutral 600
- Use: Hero descriptions, emphasis paragraphs

**Body Default**
- Font: Inter
- Size: 17px
- Line Height: 1.65
- Letter Spacing: -0.01em
- Color: Neutral 900
- Use: Main content, paragraphs

**Body Small**
- Font: Inter
- Size: 15px
- Line Height: 1.5
- Color: Neutral 600
- Use: Labels, captions, secondary text

**Body Tiny**
- Font: Inter
- Size: 14px
- Line Height: 1.5
- Color: Neutral 500
- Use: Metadata, timestamps, fine print

### Typography Best Practices

- Maximum line length: 68 characters
- Paragraph spacing: 24px
- Use medium (500) or semibold (600) for emphasis, avoid italic
- Headings should have ample whitespace above and below
- Keep line height generous (1.6-1.7) for readability

## Spacing System

### Scale

Based on 8px grid system for consistent rhythm:

- **xs** - 8px - Tight spacing within components
- **sm** - 16px - Component internal spacing
- **md** - 24px - Standard spacing between elements
- **lg** - 32px - Section spacing
- **xl** - 48px - Large section spacing
- **2xl** - 64px - Major section divisions
- **3xl** - 96px - Hero sections, major breaks

### Application

- Use consistent spacing throughout the interface
- Maintain vertical rhythm with the 8px base
- Larger spacing creates visual hierarchy
- Reduce spacing on mobile to maximize screen space

---

## Elevation & Shadows

### Shadow Scale

**XS - Subtle Depth**
- `0 1px 2px 0 rgba(10, 37, 64, 0.05)`
- Use: Subtle cards, input fields

**SM - Standard Cards**
- `0 1px 3px 0 rgba(10, 37, 64, 0.1), 0 1px 2px -1px rgba(10, 37, 64, 0.1)`
- Use: Default cards, buttons, panels

**MD - Elevated Elements**
- `0 4px 6px -1px rgba(10, 37, 64, 0.1), 0 2px 4px -2px rgba(10, 37, 64, 0.1)`
- Use: Hover states, dropdown menus

**LG - Floating Elements**
- `0 10px 15px -3px rgba(10, 37, 64, 0.1), 0 4px 6px -4px rgba(10, 37, 64, 0.1)`
- Use: Modals, popovers

**XL - Maximum Elevation**
- `0 20px 25px -5px rgba(10, 37, 64, 0.1), 0 8px 10px -6px rgba(10, 37, 64, 0.1)`
- Use: Large modals, important overlays

### Elevation Best Practices

- Use shadows sparingly for clear hierarchy
- Increase shadow on hover for interactive elements
- Combine with subtle translation for depth
- Keep shadows consistent with light source (top-down)

## Border Radius

### Scale

- **sm** - 4px - Small elements, badges
- **md** - 8px - Buttons, inputs, standard cards
- **lg** - 12px - Large cards, panels
- **xl** - 16px - Hero sections, major components
- **full** - 9999px - Pills, rounded badges

### Application

- Use medium (8px) radius for most interactive elements
- Larger radius (12-16px) for content cards
- Full radius for status badges and pills
- Consistent radius creates cohesive design

## Animation & Transitions

### Timing

- **Fast** - 150ms - Micro-interactions, button states
- **Base** - 250ms - Standard transitions, hover effects
- **Slow** - 350ms - Complex animations, transformations

### Easing

- Default: `cubic-bezier(0.4, 0, 0.2, 1)` - Natural, smooth
- Use for: All standard transitions
- Purpose: Creates polished, professional feel

### Interactive States

**Buttons**
- Hover: Scale brightness, lift with shadow
- Active: Reduce shadow, reset transform
- Focus: Show focus ring with accent color
- Transition: All properties 250ms

**Links**
- Hover: Darken color, thicken underline
- Focus: Show focus outline
- Transition: Color and decoration 150ms

**Cards**
- Hover: Lift with shadow, subtle border change
- Transition: All properties 250ms

### Best Practices

- Keep animations subtle and purposeful
- Use transitions consistently across similar elements
- Avoid animations longer than 350ms
- Disable animations for reduced-motion preferences

- Disabled: 50% opacity, no hover effects
- Loading: Show spinner, maintain dimensions

- Background: White
- Border: 1px solid Border Light
- Border Radius: 12px
- Padding: 32px
- Shadow: SM (increases to MD on hover)

#### Input Fields
- Border: 2px solid Border

- Font: Inter, 15px, weight 600

- Padding: 6px 14px

## Logo Usage

### Current Logo Files

- Primary: `/5_logo.jpg`
- Banner: `/5_banner.jpg`

### Specifications for New Logo Design

When creating new logo files, follow these specifications:

#### Logo Variants Needed

1. **Primary Full-Color Logo**
   - Horizontal layout
   - Navy blue (Primary 700) + Teal (Accent 600) combination
   - Include "5R-AI" wordmark
   - Use Lexend font family
   - Minimum size: 120px width
   - Clear space: Minimum 16px on all sides

2. **Simplified Mark**
   - Icon-only version for small applications
   - Square format
   - Minimum size: 32px × 32px
   - Works in single color

3. **White/Reverse Logo**
   - For dark backgrounds
   - All-white version
   - Maintain same proportions

4. **Monochrome Version**
   - Single color (Primary 700)
   - For print and simple applications

#### Design Principles for Logo

- **Modern & Trustworthy**: Geometric shapes, clean lines
- **Professional**: Avoid overly playful elements
- **Memorable**: Simple, distinctive mark
- **Scalable**: Works from favicon to billboard
- **Representative**: Incorporate AI or search concept subtly

#### Logo Don'ts

- Don't stretch or distort proportions
- Don't add effects (shadows, glows, gradients on wordmark)
- Don't change colors outside approved palette
- Don't place on busy backgrounds
- Don't use low-resolution files

#### File Formats Required

- SVG (vector, primary format)
- PNG (transparent background, 2x resolution)
- JPG (for photos/complex backgrounds)

## Voice & Tone

### Brand Personality

- **Professional** but approachable
- **Confident** but not arrogant
- **Clear** and straightforward
- **Trustworthy** and reliable
- **Modern** without being trendy

### Writing Style

- Use active voice
- Short sentences and paragraphs
- Clear, jargon-free language
- Speak directly to tradespeople
- Focus on practical benefits

### Examples

**Good:**
"Rex sends you one clear task each week to improve your visibility."

**Avoid:**
"Leverage our AI-powered platform to optimize your digital presence through strategic task-based interventions."

## Accessibility

### Color Contrast

All text must meet WCAG AA standards:
- Normal text: 4.5:1 minimum contrast
- Large text (18px+): 3:1 minimum contrast
- Interactive elements: Clearly distinguishable

### Focus States

- All interactive elements must have visible focus states
- Use 3px outline with Accent 500 color
- 2px offset from element
- Never remove focus indicators

### Screen Readers

- Use semantic HTML
- Provide alt text for images
- Use ARIA labels where needed
- Maintain logical heading hierarchy

### Keyboard Navigation

- All functionality available via keyboard
- Logical tab order
- Skip navigation links where appropriate
- Escape key closes modals/overlays

## Implementation Notes

### CSS Custom Properties

All design tokens are defined as CSS custom properties (variables) in `src/index.css`:

```css
var(--color-primary-600)
var(--spacing-lg)
var(--shadow-md)
var(--radius-lg)
var(--transition-base)
```

### Font Loading

Fonts are loaded via Google Fonts CDN:
- Inter (weights: 400, 500, 600, 700, 800)
- Lexend (weights: 400, 500, 600, 700)

Enable font display swap for performance.

### Responsive Design

- Mobile-first approach
- Breakpoint: 768px (tablet/desktop)
- Fluid typography and spacing
- Touch-friendly tap targets (minimum 44px × 44px)
